// 01 · contexto

O Desafio Operacional

Quando o projeto começou, toda a operação de cobrança dependia de processos manuais altamente custosos em tempo e sujeitos a erros humanos. O cenário precisava mudar.

⏱️

Alto tempo operacional

Horas diárias consumidas acessando CRM, exportando relatórios manualmente, filtrando credor e período, tratando planilhas com células mescladas.

⚠️

Inconsistências nos dados

Sem padronização, cada relatório era ligeiramente diferente. Erros de consolidação geravam divergências que comprometiam análises estratégicas.

📁

Acúmulo de arquivos locais

Centenas de planilhas Excel espalhadas por pastas locais, sem histórico centralizado, sem versionamento, difíceis de auditar.

💸

Envios Manuais e Custos

O volume crescia, mas o processo não acompanhava. Comprar licenças de BI para cada operador era inviável, o que obrigava a gestão a tirar prints diários da tela e enviar um por um no WhatsApp.

// 02 · evolução da solução

Da Planilha ao Servidor Linux

A transformação não aconteceu de uma vez. Foi uma evolução técnica deliberada, etapa por etapa, sempre com foco em impacto real na operação.

Etapa 01

Processo Manual em Excel

Ponto de partida: relatórios montados linha por linha no Excel. Tratamento manual de planilhas com células mescladas, filtros manuais por credor e período, consolidação sem padrão. Cada atualização consumia horas da equipe.

Excel Manual Alto custo operacional
Etapa 02

Padronização com Power Query

Primeiro ganho concreto: automatização do tratamento dentro do Excel via Power Query. Remoção de mesclas, padronização de colunas e estruturação dos dados para análises básicas. Redução significativa do trabalho de "limpeza" manual.

Power Query ETL parcial Padronização
Etapa 03

Migração para Power BI

Com os dados estruturados, o próximo passo foi visualização em Power BI. Porém, isso gerou um novo gargalo: o custo de licenças Pro por operador era altíssimo. A solução paliativa era a gestão tirar prints dos resultados e enviar todo dia no chat para cada operador.

Power BI DAX KPIs operacionais Dashboards interativos
Etapa 04

Criação de Banco de Dados Interno

Eliminar de vez os arquivos locais. Um banco de dados relacional passou a centralizar todo o histórico da operação — contratos, pagamentos, inadimplência, desempenho por credor. Fim dos arquivos Excel espalhados, início do dado confiável e auditável.

SQL Banco relacional Centralização de dados Histórico consolidado
Etapa 05

Automação Completa com Python

Scripts Python passaram a fazer tudo automaticamente: acessar o CRM, extrair os dados, tratar inconsistências, padronizar e inserir no banco de dados. O que levava horas passou a rodar em minutos, sem intervenção humana.

Python Pandas ETL automatizado Integração CRM → DB
Etapa 06

Dashboard Próprio e Infra 24/7

O grande salto tecnológico! Substituímos o Power BI por um Web App próprio (Dash/Python) em VPS Linux. Os operadores agora acessam com login seguro (senhas com hash) e visualizam apenas a própria produção. Fim dos custos com licenças, fim dos envios de prints, operação automatizada com systemd 24/7.

Linux systemd VPS Execução 24/7 Agendamento automático Infra

// 04 · antes vs depois

O Antes e o Depois

A diferença é objetiva.

Antes — Processo Manual
  • Exportação e filtro de relatórios na mão
  • Consolidação manual de arquivos Excel
  • Altos custos de licenças de visualização
  • Gestão obrigada a tirar prints dos resultados
  • Envio diário dos prints via WhatsApp para operadores
  • Inescalável conforme volume e equipe cresciam
Depois — Plataforma Web 100% Automatizada
  • Pipeline Python extrai, trata e salva no DB automaticamente
  • Dashboard Web Próprio, eliminando custos de licenças
  • Operadores com acesso direto via login (senhas seguras com hash)
  • Visualização isolada: cada um vê apenas os próprios dados
  • Fim total do envio manual de relatórios e prints
  • Execução 24/7 em VPS Linux — 100% escalável e seguro
